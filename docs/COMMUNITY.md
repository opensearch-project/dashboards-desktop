# Community Engagement Plan

How we build and support the OSD Desktop community.

---

## Channels

| Channel | Purpose | Link |
|---------|---------|------|
| **GitHub Issues** | Bug reports, feature requests | [issues](https://github.com/opensearch-project/dashboards-desktop/issues) |
| **GitHub Discussions** | Q&A, ideas, show & tell | [discussions](https://github.com/opensearch-project/dashboards-desktop/discussions) |
| **OpenSearch Slack** | Real-time chat, #dashboards-desktop channel | [join](https://opensearch.org/slack) |
| **OpenSearch Forum** | Long-form discussions | [forum](https://forum.opensearch.org/) |

## Monthly Office Hours

- **When:** First Thursday of each month, 10am PT / 1pm ET / 6pm UTC
- **Where:** OpenSearch Community Meeting (Zoom link in Slack)
- **Format:** 30 min — demo of latest release, Q&A, roadmap discussion
- **Recording:** Posted to YouTube and linked in Slack

## Contributor Recognition

### Monthly Spotlight

Each month, we highlight contributors in the release notes and on the forum:
- Most impactful PR
- Best first contribution
- Most helpful community member (answering questions, reviewing PRs)

### CONTRIBUTORS.md

All contributors are listed in `CONTRIBUTORS.md` (auto-generated from git history). First-time contributors get a welcome message on their PR.

### Swag

Top contributors per quarter receive OpenSearch swag (stickers, t-shirts). Nominated by maintainers.

## Issue Labels

| Label | Meaning |
|-------|---------|
| `good first issue` | Suitable for new contributors |
| `help wanted` | Maintainers welcome community help |
| `enhancement` | Feature request |
| `bug` | Confirmed bug |
| `documentation` | Docs improvement |
| `question` | Needs clarification |
| `untriaged` | Auto-applied, needs maintainer review |

## How to Get Involved

1. **Use the app** — download, try it, file issues
2. **Fix a bug** — pick a `good first issue`, submit a PR ([guide](CONTRIBUTOR-ONBOARDING.md))
3. **Write a plugin/skill/MCP server** — extend the platform ([guide](GUIDE-PLUGIN-DEVELOPMENT.md))
4. **Answer questions** — help others in Discussions and Slack
5. **Write docs** — improve guides, add examples, fix typos
6. **Spread the word** — blog about it, present at meetups, share on social media
